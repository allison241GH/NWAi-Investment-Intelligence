# NWAi DD Investment Report — Format Reference

Defines the structure, scoring rules, visual layout, and content standards for the NWAi Due Diligence
Investment Report generated by `/dd-report`. This is the scored synthesis document produced by the DD team
at the *conclusion* of diligence — after the 17-folder checklist work is complete. It is the primary
input to the `/decision` command.

**Canonical visual reference: `Synergist-DD-Investment-Report-2026-03-11.docx`**
This file is the authoritative layout template. When generating any DD report, the output MUST match
its visual structure exactly. If this file is available in the workspace, read it before generating.
If generating a new script, model the docx structure directly from this file.

---

## Output Format

- **File type:** .docx (Word document)
- **Filename:** `[Company-Name]-NWAi-DD-Report-[YYYY-MM-DD].docx`
- **Page size:** US Letter (12240 × 15840 DXA)
- **Margins:** 864 DXA all sides (0.6 inch) — exact Synergist value
- **Header/Footer offset:** 708 DXA
- **Header on every page:** "New World Angels — Confidential Investment Report" (right-aligned, gray)
- **Footer on every page:** Company name (left) | Page number (right)
- **Font:** Arial throughout. Section header rows: 12pt bold white on navy. Body: 10pt. Score number: 24pt bold.

---

## ⚠ CRITICAL: VISUAL LAYOUT — 2-COLUMN SCORED SECTION TABLE

**This is the most important specification in this file. Failure to follow it causes format drift.**

Every scored section is rendered as a **2-column table** — NOT as a paragraph followed by a score callout box.

```
┌──────────────────────────────────────────────────────┬───────┐
│ #.  SECTION TITLE  (navy bg, white bold text)        │ Score │  ← header row
├──────────────────────────────────────────────────────┼───────┤
│                                                      │       │
│  Content column (9432 DXA):                          │   3   │  ← content + score
│  • Narrative paragraphs                              │       │
│  • Inline flags (❌ ⚠ ✓)                             │  RAG  │
│  • Embedded mini-tables                              │ color │
│  • Bullet points                                     │       │
│                                                      │       │
└──────────────────────────────────────────────────────┴───────┘
```

### EXACT Column Widths (measured from Synergist source XML)
- **Content column: 9432 DXA** (~89.7% of content area)
- **Score column: 1080 DXA** (~10.3% — wide enough for a single digit only)
- **Total table width: 10512 DXA** (= PAGE_W 12240 − 2 × MARGIN 864)

### Score Cell RAG Colors (exact hex values from Synergist source XML)

| Score | Fill (background) | Text color | Label |
|-------|-------------------|------------|-------|
| 4–5   | `375623`          | `FFFFFF`   | Green |
| 3     | `FFC000`          | `1F1F1F`   | Amber |
| 1–2   | `C00000`          | `FFFFFF`   | Red   |

Score number font: **24pt bold** (= `sz` value 48 in half-points), vertically centered.

**This layout is used for EVERY scored section.** Do not use standalone paragraphs with "Score: X/5" callouts.
Do not use section banners followed by paragraph blocks. Every section must be a self-contained 2-column table.

## Document Structure (in order)

### 1 — NWAi Recommendation Banner (TOP of document, before everything)

Full-width single-cell table. Color reflects verdict:
- INVEST: Green background (`E8F5E9`), dark green text
- WATCH: Amber background, dark text
- PASS: Light red background, dark text

Contains: recommendation verdict (large bold) + one-line conditional note (italic, smaller).

### 2 — Company Header Block (no score)

Multi-row table with total width 10512 DXA:
- Row 1: "COMPANY DESCRIPTION" — navy header, full-width span
- Row 2: Company description paragraph (2–4 sentences: what they do, who they serve, problem solved)
- Rows 3+: 4-column key terms grid using columns COL1/COL2/COL3/COL4 (alternating label/value pairs)
  - Fields: Platform, Stage, HQ, Round, CEO, Raise, Founded, NWAi Check Size

Below header block: structural notes (⚠ non-standard terms, gate issues) + "Scoring: 1 (lowest) to 5 (highest)" line.

### 3 — Scored Sections (one 2-column table per section)

**Sections are NUMBERED 1–11 in the section header text** (e.g., "1.  THE PROBLEM / MARKET OPPORTUNITY").

| # | Section Header | Score Basis | Scale |
|---|----------------|-------------|-------|
| 1 | THE PROBLEM / MARKET OPPORTUNITY | Timing Score (0–5) + SAM/TAM | 1–5 |
| 2 | THE SOLUTION / PRODUCT | Moat Tier 1 (0–6) → 1–5 | 1–5 |
| 3 | AI / SOFTWARE MOAT *(AI companies only)* | Moat Tier 2 AI (0–10) → 1–5 | 1–5 |
| 4 | COMPETITION & MOAT | Competitive Risk (1–10 inverted) | 1–5 |
| 5 | GO-TO-MARKET STRATEGY | Market Risk (adoption barriers) | 1–5 |
| 6 | TEAM | Execution Risk (1–10 inverted) | 1–5 |
| 7 | TECHNOLOGY & INTELLECTUAL PROPERTY | TRL + IP Status | 1–5 |
| 8 | DEAL STRUCTURE & CAP TABLE | AutoKill gates + cap table | 1–5 |
| 9 | FINANCIALS | Financial Risk (1–10 inverted) + Bear/Base/Bull | 1–5 |
| 10 | RISK *(5 = lowest risk / best \| 1 = highest risk / worst)* | Risk avg across 5 categories | 1–5 inverted |
| 11 | EXIT STRATEGY | Exit viability + 10x return support | 1–5 |

> **Section naming:** "1.  THE PROBLEM / MARKET OPPORTUNITY" — two spaces after the number.
> **Business Model content:** Folded into FINANCIALS (pricing, unit economics) and THE SOLUTION / PRODUCT. Not a standalone section.
> **Customer Pipeline content:** Folded into THE SOLUTION / PRODUCT and COMPETITION & MOAT. Not a standalone section.
> **Omit Section 3 (AI Moat)** if the company is not AI-enabled.

### 4 — Closing Sections (no score, full-width tables)

**RECOMMENDATION table** — Full-width, navy header. Content:
- Three checkbox lines: `☒ INVEST — [amount]` / `☐ WATCH — Re-evaluate` / `☐ PASS`
- Rationale: 2–4 sentences, IC-ready, no hedging
- Numbered conditions list (if conditional)
- Theme + Lead + SMEs note (TBD if pending Dealum API)

**DD TEAM VOTES table** — 4 columns: Role | Name | Recommend | Add-On Interest

**APPENDIX A — OUTSTANDING DILIGENCE ITEMS table** — 4 columns: DD Folder / Item | Action Required | Owner | Priority
Row background: RED (`FFD7D7`) for GATE ❌ items, AMBER (`FFF3CD`) for HIGH ⚠, gray for MEDIUM/LOW.

## Score Mapping Reference

### Moat Tier 1 (0–6) → 1–5
5–6 → Score 5 | 4 → Score 4 | 3 → Score 3 | 2 → Score 2 | 0–1 → Score 1

### Moat Tier 2 AI (0–10) → 1–5
8–10 → Score 5 | 6–7 → Score 4 | 4–5 → Score 3 | 2–3 → Score 2 | 0–1 → Score 1

### Risk Scores (1–10 inverted) → 1–5
Risk 1–3 → Score 5 | Risk 4–5 → Score 4 | Risk 6–7 → Score 3 | Risk 8 → Score 2 | Risk 9–10 → Score 1

### Risk Average → Section 10 Score (inverted)
Avg < 4 → Score 5 | Avg 4–5 → Score 4 | Avg 5–6 → Score 3 | Avg 7–8 → Score 2 | Avg > 8 → Score 1

### Problem/Market Score
5: TAM $1B+, Timing ≥ 4/5 | 4: Solid market, strong timing | 3: Real but uncertain | 2: Niche/low urgency | 1: Not validated

## Inline Flag Standards

- `❌` — Hard gate failure, missing critical element, deal-breaker risk
- `⚠` — Yellow flag, non-standard term, requires verification or monitoring
- `✓` — Confirmed positive, gate pass, validated element

---

## Language Standards

- **Tone:** Direct. One finding per sentence. No corporate hedging.
- **Score display:** Large bold RAG-colored number in right column. NOT "Score: X/5" in body text.
- **Risk inverted note:** Include "(5 = lowest risk / best | 1 = highest risk / worst)" in Section 10 header.
- **Recommendation:** Lead with checkbox verdict, then dollar amount, then rationale, then conditions.
- **Confidentiality:** "New World Angels — Confidential Investment Report" on every page header.

---

## Content Sourcing Priority

1. **17-folder DD checklist findings** — primary source
2. **NWAi scoring rubrics** — all scores derived from rubric outputs, not estimated
3. **Agent research briefings** — company-researcher and competitive-intelligence outputs
4. **Prior pipeline outputs** — Screening flags, Scout verdict, DD Kickoff findings
5. **Dealum application data** — deal metadata, contact info, round details

Mark any missing field `[TO BE CONFIRMED]`. Never fabricate data or scores.

## Format Drift Incident — 2026-03-19

The STL DD Report was initially generated with the wrong visual layout (narrative paragraphs + Score callout
boxes) instead of the 2-column scored table format. Root cause: the format reference described section
*content* but not the *visual layout*. A second version was generated with the correct layout structure but
with 83%/17% column widths instead of the Synergist-spec 9432/1080 DXA, and without RAG score colors.

Both issues are now fixed and captured explicitly in this reference file.
Always read this file AND the Synergist canonical reference before generating any DD report script.
